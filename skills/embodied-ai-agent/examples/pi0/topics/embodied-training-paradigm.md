# 具身智能两阶段训练范式

## 背景与目标

`π0` 视频里最有工程意义的一段，不是模型结构本身，而是它明确强调了“预训练 + 后训练”的两阶段训练策略。这关系到具身智能系统究竟该如何积累通用能力，又如何把能力落到具体任务。

### 本文关注三件事

- 预训练阶段到底在学什么。
- 后训练阶段为什么只要少量高质量数据就能带来大幅提升。
- 为什么只做其中一个阶段都不够。

### 典型适用场景

- 多任务多机器人基础能力构建。
- 特定任务快速适配与精调。
- 希望同时兼顾泛化和专家级性能的产品路线。

## 关键问题

| 维度 | 核心问题 | 典型设计抓手 |
|---|---|---|
| 表征 | 通用能力从哪里来 | 大规模异构数据预训练 |
| 动力学 / 机制 | 任务性能如何抬起来 | 小规模高质量任务后训练 |
| 决策 / 输出 | 如何平衡泛化和精度 | 阶段分离、目标分离 |
| 泛化 | 如何跨任务迁移 | 预训练吸收多样操作模式 |
| 部署 | 如何控制数据成本 | 后训练聚焦关键任务与少量数据 |

## 方法拆解

### 一条典型技术链路

1. 先用多源、多机器人、多任务数据建立基础操作知识。
2. 让模型看到足够多的状态变化、动作模式和任务条件。
3. 再在具体任务上引入少量高质量数据做定向提升。
4. 通过后训练把“会做”推到“做得稳、做得像专家”。
5. 用实机任务验证两阶段组合的真实收益。

### 常见结构模块

- 预训练数据：开源、自产、长期累积数据。
- 任务数据：特定目标任务的高质量示范。
- 训练目标：泛化能力 vs 专家能力。
- 评估方式：跨任务表现、单任务表现、恢复能力。
- 部署环节：继续采集失败样本，反哺后训练。

### 工程上最难的三点

- 预训练数据覆盖广，但噪声大、质量不均。
- 后训练数据质量高，但采集昂贵。
- 两阶段如何衔接，直接影响是否真的能从“通用”走到“可用”。

## 当前判断

- 两阶段训练很可能会成为未来具身智能团队的数据工程常态，而不是 `π0` 的一次性技巧。
- 这套范式是否成立，最终取决于后训练数据是否能持续低成本获得。
