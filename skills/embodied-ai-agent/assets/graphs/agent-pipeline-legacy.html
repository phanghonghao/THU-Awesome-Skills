Embodied AI / Multi-Source / Wiki Pipeline

Multi-Source-Research-Agent

一个面向中文 Bilibili 内容与 GitHub 仓库的具身智能研究整理 Agent。它负责发现候选内容、提炼观点、抓取截图或仓库证据、扩展相关论文与项目，并最终沉淀为可进入团队 Wiki 的中文知识条目。

Agent 名字
embodied-ai-agent
核心语言
中文
主要来源
Bilibili 视频 / GitHub 仓库
最终目标
Wiki 化知识沉淀

执行 Pipeline

这套流程按“发现 → 抓取 → 证据抽取 → 课题路由 → 理解 → 扩展 → 整理 → 审核 → 推送”执行。推送不是默认步骤，必须等用户确认 token 和批准。

01

发现候选内容

围绕“具身智能 / VLA / 模仿学习 / 机器人操作 / 世界模型 / 强化学习”等主题，在 Bilibili 和 GitHub 中筛选候选视频与仓库。

  • 按关键词、UP 主、专题合集或 GitHub query 进行搜索
  • 判断是否与 Embodied AI 主线相关
  • 优先选有明确技术路线、项目解读或可复现实现的对象
02

抓取原始信息

抓取标题、简介、标签、发布时间、时长、视频 ID、章节时间线，或仓库描述、stars、forks、README、releases 等信息。

  • 优先使用平台原始元信息
  • 视频缺字幕时退化到“简介时间线 + 截图”
  • 仓库侧补抓 README 与 release 作为证据
03

抽取证据

视频链路按章节抽帧；GitHub 链路提取 README、release 和相关仓库，形成证据链，而不是只留文字摘要。

  • 视频按章节抽取关键截图
  • 仓库侧保留 metadata、README headings 和 release
  • 作为后续“观点是否真的在原始来源里出现过”的证据
04

课题路由

在整理前先做关键词分类，把内容分发到对应板块，而不是所有视频都堆在一个文件夹或一个总文档里。

  • 例如路由到 `VLA / WM / GR00T / ALOHA / RL`
  • 允许一个视频有主课题和次级课题
  • 后续飞书推送时可以映射到不同 Wiki 节点
05

形成内容理解

把视频压缩成一条清晰主线：它讲了什么、解决了什么问题、对技术判断有什么启发。

  • 写出一句话核心判断
  • 列出章节脉络和关键论点
  • 给每张截图补“画面信息 / 对应观点 / 为什么重要”
06

扩展相关研究

把视频里提到的项目、论文和趋势补成 Wiki 级别的参考材料，而不是停留在视频复述。

  • 补官方项目页和论文链接
  • 识别可拆分成独立 Wiki 页的主题
  • 提炼后续值得持续跟踪的问题
07

整理为 Wiki Markdown

输出应优先是“专题模板页”，视频整理页只是专题页的素材来源之一。引用与署名固定放在文末，便于后续同步到飞书知识库。

  • 专题页优先套用飞书风格模板：背景与目标 / 关键问题 / 技术地图 / 方法拆解 / 对比维度 / 仓库调研
  • 视频页则保留：核心判断 / 章节脉络 / 截图证据
  • 把视频内容继续拆成 2-3 个独立课题页，放入对应板块
  • 如果内容适合可视化，补充本地 SVG 架构图 / 流程图 / 主题拆分图
  • 文末固定 `## 参考与署名`
  • 保留视频链接、标题、UP 主、发布时间、时长和抓取说明
08

可视化增强

当文字不足以表达架构、流程或板块拆分关系时，优先生成本地 SVG 图；如果用户要做对外展示或封面，再提示是否需要通过 $ai-gen 补一版视觉图。

  • 本地知识图优先用 SVG，便于版本管理与后续修改
  • 封面图、海报图、抽象概念图不自动生成，先询问用户
  • 只有用户明确同意后，才进入 `$ai-gen`
09

人工或 Agent 复核

复核不是可选项。需要确认判断是否过度延伸、截图与论点是否对齐、参考链接是否有效。

  • 检查是否误读视频主旨
  • 检查链接、项目名和时间点
  • 判断是一篇总览页还是多个专题页
10

推送到 Lark Wiki

只有在用户明确给出目标空间或父节点 token，并再次批准后，才允许进入推送步骤。

  • 先读取或初始化 `.wiki-sync.json`
  • 展示本次新增/更新文档
  • 再次等待用户确认后执行 `lark-wiki-push`

交付内容

这套 Agent 的交付物分为“原始抓取层”“结构化知识层”“Wiki 推送层”三部分。

原始抓取层

  • `metadata.json`：标题、简介、标签、发布时间、章节信息
  • `assets/frames/*.jpg`：章节截图证据
  • `manifest.json`：本次产物索引

结构化知识层

  • `assets/templates/feishu-topic-template.md`：专题模板
  • `report.md`：单视频整理稿
  • `wiki-draft.md`：飞书专题模板版总览页
  • `topics/*.md`：拆分后的独立课题页
  • `assets/diagrams/*.svg`：本地 SVG 框架图

Wiki 推送层

  • `.wiki-sync.json`：目标空间和文档映射
  • 待推送新增/更新清单
  • 经用户批准后同步到 Lark Wiki

约束规则

内容语言

  • 默认中文输出
  • 技术名词保留英文缩写
  • 适合直接进入团队中文知识库

引用位置

  • 视频信息不能放在文首占主结构
  • 统一放在文末 `## 参考与署名`
  • 兼顾引用、署名和抓取说明

推送安全

  • 没有 token，不推
  • 没有用户批准，不推
  • 必须先展示将要推送的文档列表
当前名称：Embodied-Ai-agent。它强调这不是单篇摘要脚本，而是一个“发现内容 → 归类课题 → 生成专题模板 → 按板块推送”的知识整理代理。
