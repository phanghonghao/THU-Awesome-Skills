Parallel Research Layout

Multi-Source Research Agent

一次请求先进入统一编排层，再把 Bilibili 与 GitHub 作为并行证据通道处理，最后按课题路由汇总为 Wiki 草稿、 topic notes、SVG 图和可选的飞书推送动作。

Input

User Request

Video URL, repo URL, owner/repo, or 2-5 中文关键词。

Bilibili GitHub 关键词入口

Selection Heuristic

按匹配度、技术密度、活跃度、发布时间与是否适合拆专题，选出后续处理对象。

Orchestrator

Parallel Planner

统一编排请求，把同一个研究主题拆成两个并行来源通道，再保持同一套 topic routing 规则。

  • 当前稳定源：Bilibili + GitHub
  • 统一输出：manifest / metadata / report / wiki-draft

Source Lanes

Bilibili Lane

抓取标题、简介、UP 主、时间线、封面和章节截图，形成视频证据包。

metadata chapters frames

GitHub Lane

抓取仓库元信息、README、release、topics 与相关仓库，形成仓库证据包。

repo facts readme related repos

Cross-Source Signals

对齐关键词、方法名、课题板块与可拆分子主题，避免视频侧与仓库侧各写一套命名。

Synthesis & Output

Topic Router

先定主课题，再补相关课题，保持 `VLA / WM / GR00T / ALOHA / DiffusionPolicy / RL` 的统一落点。

Knowledge Assembly

把并行证据合流成 report、wiki-draft、topic notes 和 SVG 图，而不是单篇摘要。

Deliverables

  • `outputs/.../manifest.json`
  • `report.md` / `wiki-draft.md`
  • `topics/*.md`
  • `assets/diagrams/*.svg`

Optional Feishu Push

只在用户批准后执行，并先把本地 SVG 渲染成 PNG，再通过飞书素材上传。
